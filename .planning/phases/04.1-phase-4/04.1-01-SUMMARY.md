## Accomplishments

- Fixed PAL-03: `generateBtn` now builds a `filtered` Uint8ClampedArray before `putImageData` — pixels with alpha <= 127 become (0,0,0,0), pixels with alpha >= 128 get alpha forced to 255, eliminating the cv2.IMREAD_COLOR black-compositing bug that injected spurious black swatches into generated palettes
- Fixed PAL-04: Replaced 44-line async `applyPalette()` (fetch + preview-only) with 18-line synchronous destructive implementation: `pushHistory()` → pixel walk with alpha threshold 127 → `nearestPaletteColor()` → `flushPixels()`. Palette apply is now fully undoable (Cmd+Z)
- Added `nearestPaletteColor()` helper (squared RGB Euclidean distance, no sqrt) immediately above `applyPalette` for locality
