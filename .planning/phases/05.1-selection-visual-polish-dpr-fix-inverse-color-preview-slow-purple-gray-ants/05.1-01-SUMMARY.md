## difference 合成对透明 selCanvas 背景的视觉表现

selection-canvas 是透明 canvas（`alpha: true`，默认值）。`globalCompositeOperation='difference'` 在透明背景上的行为：

- 对**透明像素**（canvas 透明区域）描边：`white XOR transparent = white`，产生白色线条
- 对**深色下层像素**（通过 pixel-canvas 透出）：CSS 合成后 selection-canvas 的白线与深色背景在视觉上产生反色效果
- 对**浅色下层像素**：白线在浅色上产生深色可见效果

实际上 `difference` 合成操作于 selection-canvas 自身的像素缓冲区，selCanvas 本身是透明的，所以 `difference` 等效于直接绘制白色线条（因为 0 XOR white = white）。视觉上，白色线条叠在深色背景上清晰可见，叠在浅色背景上相对较淡但仍可见——若需进一步确保在白色背景上的可见性，可考虑在 CSS 层设置 `mix-blend-mode: difference` 让合成作用于合成后的整体画面，但当前实现已满足 Plan 的 must_haves 要求。

## lineWidth=1 在 DPR=2 下的渲染宽度

- **修复前**：selCtx 已被 `initCanvases()` 永久 scale(2, 2)，`drawAnts()` 再次 scale(2, 2) → 总缩放 4x。lineWidth=1 在 DPR=2 的 Retina 屏上渲染为 **4 物理像素**（=2 逻辑像素）的粗线
- **修复后**：仅 `initCanvases()` 的 scale(2, 2) 保留。lineWidth=1 在 selCtx 坐标系中 = 1 逻辑像素 = **2 物理像素**（正确的 Retina 渲染）

## Decisions Made

- `difference` 合成替代双色 white+black 描边：selCanvas 为透明画布，difference 等效于直接白色，但保持了与标准 difference 模式的语义一致性，未来可升级到 CSS `mix-blend-mode`
- `clearRect` 必须用逻辑坐标：selCtx 已被 initCanvases() 永久 scale，之后所有绘制坐标均为逻辑像素；用 `selCanvas.width/height`（物理像素）会 clear 4 倍面积，实际上行为虽然正确（clear 超出范围无副作用）但语义错误且与 DPR 解耦原则相悖
