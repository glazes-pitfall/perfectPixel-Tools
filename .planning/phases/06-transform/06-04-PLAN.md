## Context

Phase 6 验证检查点发现三个 bug，均位于 `editor.html` 的变换工具实现中。

## Objective

修复 B1 / B2 / B3，不引入行为回归。

## Task 1 — B1: 修复缩放锚点（Scale Anchor Fix）

**文件：** `editor.html`

**根因：** `tools.move.onMove` handle 分支计算出新的 `floatW/H` 后，从不更新 `floatX/Y`，
导致内容始终从左上角缩放。正确行为：被拖动的手柄对面那个点应保持不动。

**实现步骤：**

1. 在 `tools.move.onDown` 的 `handle-N` 分支中，在记录 `_origFloatPixels` 的地方同时
   记录锚点坐标：

```javascript
// Anchor fractions per handle (opposite corner/edge stays fixed)
const anchorFrac = [
  [1, 1],   // 0 TL → BR
  [0.5, 1], // 1 TC → BC-center
  [0, 1],   // 2 TR → BL
  [1, 0.5], // 3 ML → MR-center
  [0, 0.5], // 4 MR → ML-center
  [1, 0],   // 5 BL → TR
  [0.5, 0], // 6 BC → TC-center
  [0, 0],   // 7 BR → TL  ← was already correct (floatX/Y never changed)
];
const [ax, ay] = anchorFrac[handleIdx];
existingTs._dragAnchorX = existingTs.floatX + ax * existingTs.floatW;
existingTs._dragAnchorY = existingTs.floatY + ay * existingTs.floatH;
existingTs._dragAnchorFracX = ax;
existingTs._dragAnchorFracY = ay;
```

2. 在 `tools.move.onMove` handle 分支，在更新 `ts.floatW/H` 之后，立刻用锚点反推
   `floatX/Y`：

```javascript
ts.floatW = newW;
ts.floatH = newH;
// Re-anchor: keep the fixed point stationary
ts.floatX = ts._dragAnchorX - ts._dragAnchorFracX * newW;
ts.floatY = ts._dragAnchorY - ts._dragAnchorFracY * newH;
```

**验收：**
- 拖 BR 手柄 → TL 角固定不动 ✓（已有行为保持不变）
- 拖 TL 手柄 → BR 角固定不动
- 拖 TC 手柄 → 底边中点固定
- 拖 ML 手柄 → 右边中点固定

## Task 2 — B2: applyZoom 后重绘浮动内容

**文件：** `editor.html`

**根因：** `applyZoom()` 更新 CSS transform 后不调用 `_drawTransformUI()`，
导致 `selCanvas`（屏幕空间）上的浮动预览停留在旧的缩放级别坐标处。

**实现步骤：**

在 `applyZoom()` 函数末尾（`document.getElementById('zoom-display').textContent = display;` 之后）添加：

```javascript
// Redraw transform overlay when zoom changes
if (EditorState.transformState && typeof _drawTransformUI === 'function') {
  _drawTransformUI();
}
```

**注意：** `_drawTransformUI` 在 `DOMContentLoaded` 内部定义，`applyZoom` 在外部
（全局作用域）。需要将 `_drawTransformUI` 提升为模块级变量，或在 `applyZoom` 末尾
通过一个全局钩子调用。

**推荐做法：** 在 `applyZoom` 函数定义之前声明一个钩子变量：
```javascript
let _onZoomChanged = null; // hook set by DOMContentLoaded
```
在 `applyZoom` 末尾调用：
```javascript
if (_onZoomChanged) _onZoomChanged();
```
在 `DOMContentLoaded` 内部赋值：
```javascript
_onZoomChanged = () => {
  if (EditorState.transformState) _drawTransformUI();
};
```

**验收：**
- 激活变换 → 用 Ctrl+= / Ctrl+- 缩放画布 → 浮动内容和手柄跟随画布缩放保持相对位置不变

## Task 3 — B3: 画布外区域可拖回浮动内容

**文件：** `editor.html`

**根因：** `pointerdown` 只绑定在 `cursorCanvas`（覆盖像素画布范围）上。
当浮动内容被拖到画布范围外，再次点击落在 `canvas-area` 背景上，
事件无法路由到 `tools.move.onDown`，导致无法继续拖拽。

**实现步骤：**

1. 在 `activateTransform()` 末尾，将 `selCanvas`（已覆盖整个 `canvas-area`）
   的 `pointer-events` 切换为 `'auto'`，并绑定转发处理器：

```javascript
// B3 fix: capture pointer events outside cursorCanvas while transform is active
selCanvas.style.pointerEvents = 'auto';
selCanvas.addEventListener('pointerdown', _selCanvasPointerDown);
```

2. 在 `applyTransform()` 和 `cancelTransform()` 末尾，恢复 `selCanvas`：

```javascript
selCanvas.style.pointerEvents = 'none';
selCanvas.removeEventListener('pointerdown', _selCanvasPointerDown);
```

3. 实现 `_selCanvasPointerDown` 处理器（定义在 `DOMContentLoaded` 内部）：

```javascript
function _selCanvasPointerDown(e) {
  // If click lands on cursorCanvas itself, let the normal handler take it
  // (selCanvas is below cursorCanvas in z-index, so this fires only outside cursorCanvas)
  const ts = EditorState.transformState;
  if (!ts) return;
  e.preventDefault();

  const handleIdx = hitTestHandle(e.clientX, e.clientY);
  if (handleIdx !== null) {
    // Handle hit — start handle drag
    ts._origFloatPixels  = ts.floatPixels.slice();
    ts._origFloatW       = ts.floatW;
    ts._origFloatH       = ts.floatH;
    ts._dragStartScaleX  = ts.scaleX;
    ts._dragStartScaleY  = ts.scaleY;
    ts._dragMode         = 'handle-' + handleIdx;
    ts._dragStartClientX = e.clientX;
    ts._dragStartClientY = e.clientY;
    ts._dragStartFloatX  = ts.floatX;
    ts._dragStartFloatY  = ts.floatY;
    const anchorFrac = [[1,1],[0.5,1],[0,1],[1,0.5],[0,0.5],[1,0],[0.5,0],[0,0]];
    const [ax, ay] = anchorFrac[handleIdx];
    ts._dragAnchorX = ts.floatX + ax * ts.floatW;
    ts._dragAnchorY = ts.floatY + ay * ts.floatH;
    ts._dragAnchorFracX = ax;
    ts._dragAnchorFracY = ay;
  } else {
    // No handle — start move drag
    ts._dragMode         = 'move';
    ts._dragStartClientX = e.clientX;
    ts._dragStartClientY = e.clientY;
    ts._dragStartFloatX  = ts.floatX;
    ts._dragStartFloatY  = ts.floatY;
  }
  selCanvas.setPointerCapture(e.pointerId);

  // pointermove / pointerup on selCanvas forward to existing tool handlers
  function onSelMove(ev) {
    // Re-use existing onMove logic via direct call (same e object structure)
    tools.move.onMove(0, 0, ev);
  }
  function onSelUp(ev) {
    tools.move.onUp(0, 0, ev);
    selCanvas.removeEventListener('pointermove', onSelMove);
    selCanvas.removeEventListener('pointerup', onSelUp);
  }
  selCanvas.addEventListener('pointermove', onSelMove);
  selCanvas.addEventListener('pointerup', onSelUp);
}
```

**验收：**
- 浮动内容拖到画布外 → 再次点击浮动区域 → 可继续拖拽
- 浮动内容在画布外 → Apply → 内容正确写入（超出部分裁剪）
- 正常画布内操作不受影响（selCanvas 仅在 transform 激活时捕获事件）
