## Task 1 — 扩展 hitTestHandle + 光标响应

**文件：** `editor.html`

### 1a. 修改 hitTestHandle 返回结构

将返回值从 `number | null` 改为 `{type, handleIdx} | null`：

```javascript
function hitTestHandle(clientX, clientY) {
  if (!EditorState.transformState) return null;
  const ts = EditorState.transformState;
  // ... (现有坐标计算不变) ...

  const INNER = 6;   // scale zone: ±6px from handle center
  const OUTER = 20;  // rotate zone: 6~20px from handle center (corners only)

  const cornerIndices = [0, 2, 5, 7]; // TL TR BL BR

  for (let i = 0; i < handlePositions.length; i++) {
    const [hx, hy] = handlePositions[i];
    const dx = Math.abs(lx - hx);
    const dy = Math.abs(ly - hy);
    if (dx <= INNER && dy <= INNER) {
      return { type: 'scale', handleIdx: i };
    }
    // Rotation zone: only for corner handles, in the outer ring
    if (cornerIndices.includes(i) && dx <= OUTER && dy <= OUTER) {
      return { type: 'rotate', handleIdx: i };
    }
  }
  return null;
}
```

### 1b. 更新所有调用 hitTestHandle 的地方

`hitTestHandle` 被调用的地方（`onDown` + `_selCanvasPointerDown`），将：
```javascript
const handleIdx = hitTestHandle(e.clientX, e.clientY);
if (handleIdx !== null) { ... }
```
改为：
```javascript
const hitResult = hitTestHandle(e.clientX, e.clientY);
if (hitResult !== null && hitResult.type === 'scale') {
  const handleIdx = hitResult.handleIdx;
  // ... 现有缩放逻辑（handleIdx 使用方式不变）...
} else if (hitResult !== null && hitResult.type === 'rotate') {
  // → 新增旋转分支（Task 2）
}
```

### 1c. 光标更新（onCursor）

在 `tools.move.onCursor` 中实现：

```javascript
onCursor(x, y, e) {
  if (!EditorState.transformState || !e) return;
  const hit = hitTestHandle(e.clientX, e.clientY);
  if (!hit) {
    cursorCanvas.style.cursor = 'move';
    return;
  }
  if (hit.type === 'rotate') {
    cursorCanvas.style.cursor = 'crosshair';
    return;
  }
  // Scale cursor: direction depends on handle
  const scaleCursors = [
    'nw-resize', 'n-resize', 'ne-resize',
    'w-resize',              'e-resize',
    'sw-resize', 's-resize', 'se-resize',
  ];
  cursorCanvas.style.cursor = scaleCursors[hit.handleIdx] ?? 'move';
},
```

`onCursor` 需要接收原始 mouse event（`e`）以获取 `clientX/Y`。
检查现有调用处：`tools[EditorState.activeTool]?.onCursor(cx, cy)` →
改为 `tools[EditorState.activeTool]?.onCursor(cx, cy, e)`（透传事件对象）。

**注意：** 当前 `onCursor` 在 `pointermove` 中调用，`e` 是 PointerEvent，直接透传即可。

## Task 2 — 旋转拖拽逻辑

**文件：** `editor.html`

### 旋转角度计算原则（相对角度拖拽）

```
pointerdown 时：
  refAngle = atan2(cursorY - centerY, cursorX - centerX)  [单位：rad]
  startAngleDeg = ts.angleDeg  [当前已有旋转角]

pointermove 时：
  curAngle = atan2(cursorY - centerY, cursorX - centerX)
  deltaDeg = (curAngle - refAngle) * 180 / π
  ts.angleDeg = startAngleDeg + deltaDeg
  → 调用 _applyRotationPreview()
```

### 在 onDown 中添加旋转分支

在处理 `existingTs` 的代码块中，`hitResult.type === 'rotate'` 分支：

```javascript
} else if (hitResult !== null && hitResult.type === 'rotate') {
  // Compute selection center in client coords
  const pixRect = pixelCanvas.getBoundingClientRect();
  const ps = pixRect.width / EditorState.width;
  const centerClientX = pixRect.left + (existingTs.floatX + existingTs.floatW / 2) * ps;
  const centerClientY = pixRect.top  + (existingTs.floatY + existingTs.floatH / 2) * ps;

  existingTs._dragMode         = 'rotate';
  existingTs._rotCenterX       = centerClientX;
  existingTs._rotCenterY       = centerClientY;
  existingTs._rotRefAngle      = Math.atan2(e.clientY - centerClientY, e.clientX - centerClientX);
  existingTs._rotStartAngleDeg = existingTs.angleDeg;
  return;
}
```

### 在 onMove 中添加旋转分支

在 `handle-*` 分支之后：

```javascript
if (ts._dragMode === 'rotate') {
  const curAngle = Math.atan2(
    e.clientY - ts._rotCenterY,
    e.clientX - ts._rotCenterX
  );
  const deltaDeg = (curAngle - ts._rotRefAngle) * 180 / Math.PI;
  ts.angleDeg = ts._rotStartAngleDeg + deltaDeg;

  // Sync Angle° input
  const aEl = document.getElementById('opt-rotate-angle');
  if (aEl) aEl.value = Math.round(ts.angleDeg);

  // Live RotSprite preview
  _applyRotationPreview();
  return;
}
```

### RotSprite 大小限制检查（已有，确认路径一致）

`_applyRotationPreview()` 内已有 128×128 限制检查 (`origBbox.w * origBbox.h > 128*128`)，
旋转拖拽也走同一路径，无需额外处理。
