# Phase 4.2 — 色卡面板 UI 整理：设计决策

## 目标

重构色卡限制面板（`#paletteSection`）的 UI 结构，使其更直观、更紧凑。这是纯前端改动，不涉及 Flask 后端。

## 最终 UI 设计（已确认）

```
┌────────────────────────┐
│ 色卡限制             ▼ │  ← 仅 chevron，无 toggle
├────────────────────────┤  ┐
│ 当前色卡  N色   [导出▼]│  │
│ 双击色块可更改          │  │ ← 小字提示
│ □□ □□ □□ □□ □□ □□ □□  │  │
│ □□ □□ □□ □□ □□ □□ [+] │  │ ← + 色块（灰底加号，点击加入前景色）
├────────────────────────┤  │
│ [未命名色卡       ▼][💾]│  │ ← combobox：命名 + 选择已保存 + 保存
│   (展开后列表底部有上传)│  │
├────────────────────────┤  │ 可
│ 颜色数量 ──●── 16      │  │ 滚
│ 算法 [快速八叉树   ▼]  │  │ 动
│ (boost 参数，条件显示) │  │
│ [      生成色卡     ]  │  │
└────────────────────────┘  ┘
╔════════════════════════╗
║ 映射 [向量匹配     ▼]  ║  ← position:sticky bottom:0
║ [     应用色卡     ]   ║
╚════════════════════════╝
```

## 决策清单

### D1：删除顶部开关

- **决定**：移除 `#paletteEnabled` toggle switch（HTML + JS）
- **原因**：该开关原为对比预览面板服务，Phase 04.1-02 已删除预览面板，开关失去语义
- **影响**：删除 `paletteEnabled` 的 `change` 事件监听器；`applyBtn.disabled` 门控改为仅依赖 `currentPalette.length > 0 && EditorState.pixels != null`

### D2：版块顺序重排

从上到下：
1. 当前色卡（swatches + export + hint）
2. Combobox（命名 / 选择 / 保存）
3. 生成区（颜色数量 + 算法 + 生成按钮）
4. **固定底部**：映射模式 + 应用色卡

原有的「tab-generate / tab-upload」结构**废弃**，内容合并进线性布局。

### D3：加号色块（替代"添加颜色"按钮）

- **样式**：与 `.palette-swatch` 完全一致的尺寸，灰色背景（`#555`），白色「+」文字
- **CSS 类**：`.add-swatch`
- **位置**：`swatchesGrid` 末尾，由 `renderSwatches()` 动态插入（每次重渲染时追加）
- **行为**：click → 读取 `EditorState.foregroundColor`（`[r,g,b,a]`） → push `[r,g,b]` 到 `currentPalette` → `renderSwatches()`
- **不打开**颜色编辑弹窗（加入后用户可双击再改）

### D4：Combobox（命名 + 选择 + 保存）

- **HTML 结构**：`<input id="paletteName">` + `<button id="palDropBtn">▼</button>` + `<button id="savePaletteBtn">💾</button>`
- **下拉列表**：自定义 div（复用现有 `#savedPalettesDropdown` 逻辑），列表底部追加一条「📁 从本地上传」选项
- **选择行为**：点击列表中某条 → 自动加载该色卡（合并原「加载」按钮行为）+ 把名字填入 input
- **保存行为**：点 💾 → 取 input 当前值作为 key 保存（与现有 savePaletteBtn 逻辑一致）
- **上传入口**：列表底部选项点击 → 触发 `#palFileInput` click（复用现有上传逻辑）
- **移除**：独立的「加载」按钮、「删除」按钮暂不做（超出本 phase 范围）

### D5：导出按钮

- **位置**：「当前色卡」行右侧，一个 `[导出▼]` 按钮，点击展开三个选项（.act / .gpl / .pal）
- **实现**：小型下拉菜单，复用现有 `exportAct/exportGpl/exportPal` 点击处理逻辑

### D6：映射模式改为 select 下拉框

- **原**：3 个 radio card（`.mode-option` + `.mode-group`）
- **新**：`<select id="mappingModeSelect">` 含 3 个 `<option>`（vector/perceptual/swap）
- **JS**：`applyPalette()` 读取 `mappingModeSelect.value` 替代原 `input[name=mappingMode]:checked`

### D7：固定底部区（映射 + 应用）

- **CSS**：在 `#paletteBody` 内用 flex column 布局；底部区域 `flex-shrink:0`；可滚动区域 `overflow-y:auto; flex:1`
- **或**：`position:sticky; bottom:0; background:var(--surface)` 方案（哪种更干净由实现决定）

## 不在本 Phase 范围内

- 色卡删除功能（可在后续 todo 中补充）
- 色卡排序 / 重命名
- 色卡分享或云端保存

## 代码约束

- 仅修改 `editor.html`（单文件）
- 可直接复用的现有函数：`renderSwatches()`、`savePaletteBtn` 点击逻辑、`loadPaletteFromLocal()`、`exportAct/Gpl/Pal` 处理逻辑、`palUploadZone` 文件处理、`palShowStatus()`
- 现有 `applyPalette()` 函数逻辑不变，只改模式读取方式
- `applyBtn.disabled` 门控：去掉 `paletteEnabled.checked` 依赖，改为 `!EditorState.pixels || currentPalette.length === 0`
