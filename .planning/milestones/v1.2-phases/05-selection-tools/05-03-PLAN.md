```javascript
function isSelectedPixel(x, y) {
  if (!EditorState.selectionMask) return true;  // no selection → all pixels writable
  const bb = EditorState.selection;
  if (!bb || x < bb.x || x >= bb.x + bb.w || y < bb.y || y >= bb.y + bb.h) return false;
  return EditorState.selectionMask[x + y * EditorState.width] === 1;
}
```

IMPORTANT: The BFS traversal itself (which pixels to visit) must NOT be restricted by the selection mask — only the final `setPixel` call is guarded. The BFS must still explore all connected matching pixels, otherwise flood fill will fail to reach pixels on the far side of a narrow selection boundary. Only the write is clipped, not the traversal.

ALSO: For the BFS `while` loop — the BFS should still explore and add neighbors regardless of selection (to handle connected fills that loop around), but `setPixel` is the gated operation. The visited check (`if (!visited[ni] && matches(nx, ny))`) remains unchanged.

**1. Add `deleteSelection()` and `fillSelection()` helper functions** — add these as top-level functions near the other selection helpers (near `invertSelection`):

```javascript
function deleteSelection() {
  if (!EditorState.selectionMask) return;
  pushHistory();  // bucket model: push before writes
  const W = EditorState.width;
  for (let i = 0; i < EditorState.selectionMask.length; i++) {
    if (EditorState.selectionMask[i]) {
      const x = i % W, y = (i / W) | 0;
      setPixel(x, y, [0, 0, 0, 0]);  // transparent — global constraint allows alpha=0
    }
  }
  flushPixels();
}

function fillSelection() {
  if (!EditorState.selectionMask) return;
  pushHistory();  // bucket model: push before writes
  const W = EditorState.width;
  const color = [...EditorState.foregroundColor];
  color[3] = 255;  // enforce fully opaque (global tool output constraint)
  for (let i = 0; i < EditorState.selectionMask.length; i++) {
    if (EditorState.selectionMask[i]) {
      const x = i % W, y = (i / W) | 0;
      setPixel(x, y, color);
    }
  }
  flushPixels();
}
```
