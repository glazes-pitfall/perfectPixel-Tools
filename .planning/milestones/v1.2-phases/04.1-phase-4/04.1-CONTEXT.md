# Phase 04.1: Phase 4 返工 - Context

**Gathered:** 2026-03-03

<domain>
## Phase Boundary

本 Phase 修正 Phase 4 实现中的两个明确缺陷，不新增功能：

1. **Alpha 分类修正**：色卡生成时错误地将半透明像素（如 rgba(0,0,0,64)）计入颜色统计，导致产生虚假黑色等色号。需按 alpha 阈值（128）正确过滤。

2. **色卡应用模式改变**：将"应用色卡"从非破坏性预览（不触碰 EditorState.pixels）改为破坏性直接应用（写入 EditorState.pixels + pushHistory），并移除右侧预览对比面板。

</domain>

<decisions>
## Implementation Decisions

### Alpha 分类规则（PAL-03）— 用户锁定

- **Alpha = 255（纯色）**：正常计入色卡，按现有逻辑处理。无变化。
- **128 ≤ alpha < 255（不透明一侧）**：视作不透明像素，以当前 RGB 值参与色卡颜色统计与最近色匹配。应用时输出 alpha = 255。
- **1 ≤ alpha ≤ 127（透明一侧）**：视作透明像素。**不计入色卡颜色统计**，应用色卡时替换为纯透明 (0, 0, 0, 0)。
- **Alpha = 0（纯透明）**：不计入色卡，应用时替换为纯透明。

**隐式透明槽**：alpha ≤ 127 的像素统一由隐式规则处理（→ 纯透明），不占用用户设置的色卡颜色数量上限，也不显示在 UI 色卡条目中。

### 应用色卡（PAL-04）— 用户锁定

- 用户点击"应用色卡"按钮后，立即执行 `pushHistory()`，然后遍历 `EditorState.pixels` 执行替换，最后调用 `flushPixels()` 重绘。
- 替换规则：
  - pixel.alpha ≥ 128 → 找最近色卡色，输出该颜色（alpha = 255）
  - pixel.alpha ≤ 127 → 输出 (0, 0, 0, 0)
- 操作完成后画布直接更新，无弹窗、无对话框。
- 该操作计入撤销历史：Cmd+Z 可完全还原。

### 移除 palette-result-panel — 用户锁定

- `#palette-result-panel` 元素及相关 HTML/CSS/JS 全部从 `editor.html` 删除。
- `#canvas-area` 不再需要 `flex-row` 承载右侧预览面板（如有此布局可回退，按实际情况决定）。
- 所有与 `palette-result-panel` 相关的 JS 逻辑（beforeCanvas/afterCanvas 的 diff 渲染）一并删除。

### Claude's Discretion

- 最近色匹配算法：与现有 `applyPalette` 逻辑保持一致（基于欧氏距离 RGB，或现有实现方式）。
- 按钮标签文字：保持现有"应用色卡"文字，无需改变。
- 色卡生成 (`generatePalette`) 的 alpha 过滤：只过滤参与 k-means/颜色量化的候选像素列表，不修改底层量化算法。

</decisions>

<specifics>
## Specific Requirements

```
// 色卡生成：候选像素过滤
function isOpaqueForPalette(r, g, b, a) {
  return a >= 128;  // alpha ≤ 127 → 忽略
}

// 应用色卡：单像素替换逻辑
function applyPaletteToPixel(r, g, b, a, palette) {
  if (a <= 127) return [0, 0, 0, 0];  // 透明一侧 → 纯透明
  return [...nearestPaletteColor(r, g, b, palette), 255];  // 不透明一侧 → 最近色
}

// 应用色卡按钮 handler
applyBtn.addEventListener('click', () => {
  if (!EditorState.pixels || !EditorState.palette.length) return;
  pushHistory();  // save-before（应用是瞬时操作）
  applyPaletteToCanvas(EditorState.palette);
  flushPixels();
  syncColorUI();  // optional: sync current color if needed
});
```

### 关键文件位置
- 全部实现在 `editor.html` 中（单文件架构）
- 色卡生成相关函数：grep `generatePalette` 或 `generateBtn`
- 应用色卡相关函数：grep `applyPalette` 或 `applyBtn`
- palette-result-panel：grep `palette-result-panel`
